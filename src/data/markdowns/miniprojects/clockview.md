# Analog Clock View

**Analog Clock View** is a custom clock component built entirely with **Jetpack Compose Canvas**. Every visual element—including the clock face, hour markers, minute markers, and clock hands—is procedurally rendered using Compose's drawing APIs. Rather than relying on static images or vector assets, the clock dynamically calculates hand positions based on the supplied time and renders them using geometric transformations.

The project demonstrates how mathematical calculations and Canvas rendering can be combined to build reusable, responsive UI components.

---

# Features

## Fully Custom Clock Face

The entire clock is rendered from drawing primitives.

```kotlin
Canvas {

    drawClockOutlines()

    drawHourHandle(...)

    drawMinuteHandle(...)

    drawSecondHandle(...)
}
```

No bitmap images or SVG assets are used—every element is generated programmatically.

---

# Time-Based Hand Calculations

Each clock hand is positioned using mathematical angle calculations derived from the current time.

```kotlin
val secondDegree =
    -90f + second * 6f

val minuteDegree =
    -90f + minute * 6f +
    second * 0.1f

val hourDegree =
    -90f +
    (hour * 5) * 6f +
    minute * 0.5f
```

These calculations ensure that each hand moves naturally and continuously rather than jumping between positions.

---

# Smooth Hour & Minute Movement

The hour and minute hands account for the progress of smaller time units.

For example:

- Minute hand responds to seconds.
- Hour hand responds to minutes.

This produces a more realistic analog clock animation.

---

# Rotational Transformations

Each clock hand is rendered using Canvas rotation.

```kotlin
rotate(degree) {

    drawLine(...)

}
```

Drawing a single horizontal line and rotating it around the clock center greatly simplifies the implementation.

---

# Procedural Tick Marks

The clock face is generated by rotating drawing operations sixty times.

```kotlin
repeat(60) {

    rotate(it * 6f) {

        drawLine(...)

    }
}
```

Each iteration represents one minute marker around the circumference.

---

# Major and Minor Markers

The implementation distinguishes between:

- Minute markers
- Hour markers
- Quarter-hour markers

```kotlin
if(it % 15 == 0) {

    ...

}
else if(it % 5 == 0) {

    ...

}
```

Different lengths and colors improve readability while adding visual hierarchy.

# Responsive Canvas Rendering

All dimensions are calculated relative to the Canvas size.

```kotlin
radius = size.width / 2
```

This allows the clock to scale seamlessly across different device sizes and layouts.

---

# Modular Drawing Architecture

Rendering responsibilities are divided into reusable drawing functions.

```kotlin
drawClockOutlines()

drawHourHandle()

drawMinuteHandle()

drawSecondHandle()
```

Separating each drawing operation improves readability, maintainability, and reusability.

---

# Pure Vector Rendering

The clock consists entirely of:

- Lines
- Circles
- Rotations
- Stroke rendering

This keeps rendering lightweight while maintaining crisp visuals at any resolution.
